6.1. Команда mount

**Назначение:** монтирование файловых систем (подключение устройств к дереву каталогов).

**Просмотр смонтированных файловых систем:**

**Просмотр использования дисков:**

**Просмотр файла /etc/fstab (таблица монтирования):**

**Теоретическое описание монтирования (для справки):**

Команда mount используется для подключения файловой системы устройства к дереву каталогов. Общий формат:

bash

mount \[-t тип_фс\] \[-o опции\] устройство точка_монтирования

Примеры (теоретически):

- sudo mount /dev/sdb1 /mnt/usb — монтирование устройства в каталог /mnt/usb
- sudo mount -t vfat /dev/sdb1 /mnt/usb — монтирование с указанием типа FAT32
- sudo mount -o ro /dev/sdb1 /mnt/usb — монтирование в режиме только чтения
- sudo umount /mnt/usb — демонтирование

6.2. Команда fsck

**Назначение:** проверка и восстановление целостности файловой системы (file system check).

**Формат команды:**

bash

fsck \[-опции\] устройство

**Основные опции:**

- \-y — автоматически отвечать "yes" на все вопросы
- \-n — только проверка без исправлений
- \-f — принудительная проверка даже если ФС чистая
- \-t — указание типа файловой системы
- **Примеры (теоретически):**
- bash
- sudo fsck /dev/sda1 # проверка раздела
- sudo fsck -y /dev/sda1 # проверка с автоматическим исправлением
- sudo fsck -n /dev/sda1 # только проверка без исправлений
- **Важно:** перед проверкой файловая система должна быть размонтирована или находиться в режиме только чтения.

6.3. Команда mkfs

**Назначение:** создание файловой системы на устройстве (форматирование).

**Формат команды:**

bash

mkfs.тип_фс \[-опции\] устройство

**Основные типы файловых систем:**

- mkfs.ext4 — для создания ext4
- mkfs.vfat — для создания FAT32
- mkfs.xfs — для создания XFS
- mkfs.ntfs — для создания NTFS
- **Примеры (теоретически):**
- bash
- sudo mkfs.ext4 /dev/sdb1 # создание ext4
- sudo mkfs.vfat -F 32 /dev/sdb1 # создание FAT32
- sudo mkfs.ext4 -L "DATA" /dev/sdb1 # создание с меткой

**Внимание:** команда mkfs удаляет все данные на устройстве!

# \## Ответы на контрольные вопросы (Лабораторная работа № 5)

\---

\### 1. Какие команды используются для создания, копирования, перемещения и удаления файлов?

| Действие | Команда | Пример |

|----------|---------|--------|

| Создание файла | \`touch\` | \`touch file.txt\` |

| Копирование | \`cp\` | \`cp file1.txt file2.txt\` |

| Перемещение/переименование | \`mv\` | \`mv old.txt new.txt\` |

| Удаление файла | \`rm\` | \`rm file.txt\` |

| Удаление каталога | \`rmdir\` (пустой) или \`rm -r\` (с содержимым) | \`rmdir dir\` или \`rm -r dir\` |

\---

\### 2. Как просмотреть содержимое текстового файла?

| Команда | Назначение | Пример |

|---------|------------|--------|

| \`cat\` | Вывод всего файла целиком | \`cat file.txt\` |

| \`less\` | Постраничный просмотр с навигацией | \`less file.txt\` |

| \`head\` | Вывод первых N строк (по умолчанию 10) | \`head -5 file.txt\` |

| \`tail\` | Вывод последних N строк (по умолчанию 10) | \`tail -3 file.txt\` |

\*\*Управление в less:\*\*

\- \`Space\` — следующая страница

\- \`Enter\` — следующая строка

\- \`b\` — предыдущая страница

\- \`q\` — выход

\---

\### 3. Что такое права доступа? Как они интерпретируются для файлов и каталогов?

Права доступа определяют, какие действия могут выполнять пользователи с файлами и каталогами.

| Право | Обозначение | Для файла | Для каталога |

|-------|-------------|-----------|--------------|

| Чтение | \`r\` | Просмотр содержимого | Просмотр списка файлов (\`ls\`) |

| Запись | \`w\` | Изменение, удаление | Создание/удаление файлов |

| Выполнение | \`x\` | Запуск программы | Вход в каталог (\`cd\`) |

\*\*Пример:\*\* \`-rwxr-xr--\` — это файл (\`-\`), владелец имеет все права (\`rwx\`), группа — чтение и выполнение (\`r-x\`), остальные — только чтение (\`r--\`).

\---

\### 4. Какие существуют способы изменения прав доступа?

\*\*1. Символьный способ:\*\*

\`\`\`bash

chmod u+x file # добавить выполнение для владельца

chmod g-w file # убрать запись для группы

chmod o=r file # установить только чтение для остальных

chmod a+rx file # добавить чтение и выполнение для всех

\`\`\`

\*\*2. Цифровой (восьмеричный) способ:\*\*

| Цифра | Права |

|-------|-------|

| 7 | rwx |

| 6 | rw- |

| 5 | r-x |

| 4 | r-- |

| 3 | -wx |

| 2 | -w- |

| 1 | --x |

| 0 | --- |

\`\`\`bash

chmod 755 file # rwxr-xr-x

chmod 644 file # rw-r--r--

chmod 700 file # rwx------

\`\`\`

\`\`\`bash

\### 8. Основные возможности команды mv

Команда \`mv\` (move) используется для:

\- \*\*Переименования\*\* файлов и каталогов

\- \*\*Перемещения\*\* файлов и каталогов в другое место

\`\`\`bash

\# Переименование

mv oldname.txt newname.txt

\# Перемещение

mv file.txt /path/to/directory/

\# Перемещение с переименованием

mv file.txt /path/to/directory/newname.txt

\# Перемещение каталога

mv dir1 /path/to/dir2/

\# С запросом подтверждения при перезаписи

mv -i file.txt /path/to/directory/

\`\`\`

\### 9. Что такое права доступа? Как они могут быть изменены?

\*\*Права доступа\*\* — это механизм защиты, определяющий, кто и какие действия может выполнять с файлом или каталогом. Права разделены на три категории:

\- \`u\` (user) — владелец

\- \`g\` (group) — группа

\- \`o\` (others) — остальные

\*\*Изменение прав:\*\*

\`\`\`bash

\# Символьный способ

chmod u+x file # добавить выполнение владельцу

chmod go-w file # убрать запись у группы и остальных

chmod a=r file # установить только чтение для всех

\# Цифровой способ

chmod 755 file # rwxr-xr-x

chmod 644 file # rw-r--r--

chmod 700 file # rwx------

\`\`\`
